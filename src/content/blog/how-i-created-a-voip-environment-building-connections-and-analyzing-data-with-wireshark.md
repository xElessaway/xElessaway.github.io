<p>Hey folks, it’s Ahmed Mahmoud, or you might know me as xElessaway. Today, I want to share my journey of setting up a VoIP Environment and capturing its essence with Wireshark. All it takes is a virtual machines (VMs) to kickstart your VoIP server. I personally went with Asterisk for the server side and MicroSIP for Clients calls. And if you’re more inclined towards using your Android device, you can even try out ‘MizuDroid.’ Excited to get started? Let’s jump into the essentials for getting your server up and running.</p><p>Tools You’ll Need</p><ul><li><a href="https://www.asterisk.org/">Asterisk “VoIP Server”</a></li><li><a href="https://www.microsip.org/">MicroSIP</a></li><li><a href="https://www.vmware.com/">VMware</a> or <a href="https://www.virtualbox.org/">VMBox</a></li><li><a href="https://play.google.com/store/apps/details?id=com.mizuvoip.mizudroid.app&amp;hl=ar&amp;gl=US">MizuDroid</a></li></ul><p>But before we dive into the setup, let’s understand the fundamentals of VoIP Protocol.</p><p>VoIP, or Voice over Internet Protocol, is a technology that allows you to make voice calls using a broadband internet connection instead of a regular (or analog) phone line. It converts the audio signal, which is analog, into digital data that can be transmitted over the internet.</p><figure><img loading="lazy" referrerpolicy="no-referrer" alt="" src="https://cdn-images-1.medium.com/max/981/0*aLwXA2tB45uOJPr5.png" /><figcaption>VoIP RTP Packets Movement</figcaption></figure><p>So, let’s head to our ubuntu server on the VM and download Asterisk</p><pre>wget https://downloads.asterisk.org/pub/telephony/asterisk/asterisk-20-current.tar.gz | tar -zxvf asterisk-20-current.tar.gz</pre><p>now you need to install some necessary libraries let’s run the command.</p><pre>sudo apt-get install build-essential libssl-dev libncurses5-dev libnewt-dev libxml2-dev libsqlite3-dev uuid-dev libedit-dev libjansson-dev</pre><figure><img loading="lazy" referrerpolicy="no-referrer" alt="" src="https://cdn-images-1.medium.com/max/1024/1*6JFIMaYAp7-pb0uMOUgmSQ.png" /><figcaption>Asterisk Files</figcaption></figure><p>Alright, let’s kick off the Asterisk installation with ‘make,’ but before diving into it, let’s ensure that SIP (Session Initiation Protocol), the backbone of VoIP for voice and audio communication, is enabled. After all, VoIP is all about voice and audio, right? Let’s make sure it’s activated and ready to go :D!</p><pre>sudo make menuselect</pre><figure><img loading="lazy" referrerpolicy="no-referrer" alt="" src="https://cdn-images-1.medium.com/max/1024/1*8M5qFANBvO-DLFDt9MG3Dg.png" /><figcaption>1st Phase of menuselect</figcaption></figure><p>When configuring the Channel Drivers, make sure to enable ‘chan_sip.’ This step ensures that SIP, the cornerstone of VoIP communication, is properly configured and ready to facilitate seamless connections.</p><figure><img loading="lazy" referrerpolicy="no-referrer" alt="" src="https://cdn-images-1.medium.com/max/1024/1*L7u4RB-JYCpLLfnFJvbBBw.png" /><figcaption>Enable SIP Chain</figcaption></figure><p>After enabling ‘chan_sip’ and saving the changes, let’s proceed by running the configure file and initiating the installation process. This step will configure Asterisk according to our settings and install it onto our system.</p><pre>#!bin/bash<br>./configure<br>sudo make<br>sudo make install<br>sudo make samples</pre><p>And we wait and wait and wait.</p><figure><img loading="lazy" referrerpolicy="no-referrer" alt="" src="https://cdn-images-1.medium.com/max/735/0*vCMorAZt8_fZ37GW.jpg" /></figure><p>And finally, we’ve completed the setup process. Now, let’s move on to configuring usernames and passwords. We’ll focus on three files located in “/etc/asterisk” directory and update them accordingly.</p><p><strong>sip.conf</strong></p><pre><br>[general]<br>context=internal<br>allowguest=no<br>allowoverlap=no<br>bindport=5060<br>bindaddr=0.0.0.0<br>srvlookup=no<br>disallow=all<br>allow=ulaw<br>alwaysauthreject=yes<br>canreinvite=no<br>nat=yes<br>session-timers=refuse<br>localnet=192.168.0.0/255.255.255.0<br><br>[1234]#Client 1<br>type=friend<br>host=dynamic<br>secret=1234<br>context=internal<br><br>[4321]#Client 2<br>type=friend<br>host=dynamic<br>secret=4321<br>context=internal<br><br></pre><blockquote>extensions.conf</blockquote><pre>[internal]<br>exten =&gt; 1234,1,Answer()<br>exten =&gt; 1234,2,Dial(SIP/1234,60)<br>exten =&gt; 1234,3,Playback(vm-nobodyavail)<br>exten =&gt; 1234,4,VoiceMail(1234@main)<br>exten =&gt; 1234,5,Hangup()<br><br>exten =&gt; 4321,1,Answer()<br>exten =&gt; 4321,2,Dial(SIP/4321,60)<br>exten =&gt; 4321,3,Playback(vm-nobodyavail)<br>exten =&gt; 4321,4,VoiceMail(4321@main)<br>exten =&gt; 4321,5,Hangup()<br><br>exten =&gt; 8001,1,VoicemailMain(1234@main)<br>exten =&gt; 8001,2,Hangup()<br><br>exten =&gt; 8002,1,VoicemailMain(4321@main)<br>exten =&gt; 8002,2,Hangup()</pre><blockquote>voicemail.conf</blockquote><pre>[main]<br>1234 =&gt; 1234<br><br>4321 =&gt; 4321</pre><p>Now that we’ve set up the two clients ready to call each other, let’s ensure Asterisk loads the ‘chan_sip’ module. We can do this by opening the ‘module.conf’ file and updating a specific line.</p><figure><img loading="lazy" referrerpolicy="no-referrer" alt="" src="https://cdn-images-1.medium.com/max/498/1*ChD1ywUMrvoiCHJV3QGFvw.png" /><figcaption>autoload in module.conf</figcaption></figure><p>Make sure autoload is yes then go to line noload = chan_sip.so</p><figure><img loading="lazy" referrerpolicy="no-referrer" alt="" src="https://cdn-images-1.medium.com/max/727/1*bPTif2ZoBP45Dmxss931_g.png" /><figcaption>load the chain_sip.so</figcaption></figure><p>and make it</p><blockquote>;loadload = chan_sip.so</blockquote><p>Now we are done. let’s try to run the Asterisk server.</p><pre>sudo asterisk -r</pre><figure><img loading="lazy" referrerpolicy="no-referrer" alt="" src="https://cdn-images-1.medium.com/max/809/1*6yTFZ4LaXujmWWdvlIUKFg.png" /><figcaption>asterisk.ctl exist issue</figcaption></figure><p>If you got faced this issue you can split the terminal and run</p><pre>sudo asterisk -c</pre><p>then run the second command again.</p><figure><img loading="lazy" referrerpolicy="no-referrer" alt="" MicroSIP.</p><figure><img loading="lazy" referrerpolicy="no-referrer" alt="" src="https://cdn-images-1.medium.com/max/471/1*3zUq5xXFU0CD7eAkxCAY5g.png" /><figcaption>Add account to
